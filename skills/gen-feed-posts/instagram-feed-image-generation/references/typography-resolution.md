# Typography Resolution

`typography.*.style` ごとの文言生成方針。

## cover-main

- アプリの本質を一言で伝える
- 20文字以内を優先
- まず push の catchcopy を見る
- なければ product summary を短文化する

## feature-headline

- 何が見返せるか、何ができるかを短く言う
- 24文字以内を優先
- main features を束ねて言い換える

## benefit-headline

- 使うと何が残るか、どう変わるかを前に出す
- 機能名より価値を優先する

## feature-label

- 2枚目で使う短い機能文
- 12文字以内を優先
- `main_features.title` を短くし、必要なら description から動詞を借りる

## short-support

- 見出しの意味を補う短文
- 18文字以内を優先
- 説明しすぎない
