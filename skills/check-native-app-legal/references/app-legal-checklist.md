# アプリ法務チェック

この分類は一次確認の入口です。コードと公開ページの両方を照合して初めて判断できる項目は、最終的に「人間がチェックする必要のあるもの」でも再確認します。

## 1. アプリコードを読み込めば確認しやすいもの

- [ ] アプリ内からプライバシーポリシーへ到達できる導線がある
- [ ] 取得・アクセス・送信する利用者情報を、連絡先、位置情報、端末ID、広告ID、購入履歴、利用履歴、診断情報、写真・音声・ユーザー投稿などの粒度で棚卸しできる
- [ ] サードパーティSDK、広告SDK、分析ツール、クラッシュ解析、ログ基盤などの導入有無を確認できる
- [ ] Appleのプライバシーマニフェストで、アプリ本体と第三者SDKの収集データ、理由が必要なAPIの利用目的が宣言されている
- [ ] IDFA、AAID、端末識別子、Cookie相当の識別子、トラッキング用途SDK、ATTや同意UIの実装有無を確認できる
- [ ] 位置情報、連絡先、写真、カメラ、マイク、ヘルスケア、金融情報などのセンシティブ権限の宣言と利用箇所を確認できる
- [ ] 権限要求の直前に、ユーザー向けの目的説明画面や文言が実装されている
- [ ] アカウント作成機能がある場合、アプリ内からアカウント削除を申請できる導線がある
- [ ] アカウント削除APIや削除フローが、関連ユーザーデータ削除または保持理由の分岐を扱っている
- [ ] 年齢確認、子ども向け表示、保護者同意、子ども向けSDK制限、広告制限に関係する実装があるか確認できる
- [ ] 利用規約への同意取得タイミングが、初回起動、会員登録、課金前などの重要地点で実装・記録されている
- [ ] ユーザー投稿、画像、文章、レビュー、プロフィール等を扱う場合、削除、通報、ブロック、モデレーションの導線がある
- [ ] サブスクリプション、無料トライアル、アプリ内課金、外部決済リンクの表示文言、価格、期間、自動更新、解約導線が実装上確認できる
- [ ] 広告、アフィリエイト、スポンサー表示、ネイティブ広告、リワード広告のSDKや表示導線が確認できる
- [ ] レビュー誘導、ランキング操作、誇大広告、誤認表示、ダークパターンになり得る導線が実装されていないか確認できる
- [ ] 通信暗号化、保存時暗号化、認証、アクセス制御、ログのマスキング、脆弱性対応、インシデント対応に関係する実装や設定を確認できる
- [ ] App Store審査 / Google Play審査用に必要なデモモード、制限機能の説明、レビュー用アカウント導線が実装または設定されている

## 2. App Store / Google Play / 公開ページから確認できるもの

- [ ] App Store Connect / Play Console に登録するプライバシーポリシーURLが公開状態で、アプリ固有の内容になっている
- [ ] Google Play向けのプライバシーポリシーは、公開URLで、非PDF・非編集可能・地域制限なしで閲覧できる
- [ ] プライバシーポリシーに、取得する各データの利用目的、取得方法、共有先、保存期間、削除方針、問い合わせ先が記載されている
- [ ] サードパーティSDK、広告SDK、分析ツール、クラッシュ解析、ログ基盤によるデータ収集・共有がプライバシーポリシーで開示されている
- [ ] App Storeのプライバシー詳細、プライバシーラベル、Google Playのデータセーフティ申告が入力・公開されている
- [ ] アカウント作成機能がある場合、アプリ外からのアカウント削除申請ページが公開され、Play Console等に登録されている
- [ ] アカウント削除時に削除するデータと、法令・不正対策・会計などの理由で保持するデータが公開ページで説明されている
- [ ] 利用規約ページに、利用条件、禁止事項、アカウント停止、サービス変更・終了、免責、損害賠償、準拠法、問い合わせ先が記載されている
- [ ] ストア掲載ページのアプリ名、アイコン、説明文、スクリーンショット、カテゴリ、年齢レーティングが実態と一致している
- [ ] ストア掲載ページやPlay Consoleで、広告有無、対象年齢、コンテンツレーティング、ニュースアプリ等の必要申告が完了している
- [ ] 課金、サブスクリプション、無料トライアル、返金、解約、外部決済リンクに関するストア表示や公開説明が明確になっている
- [ ] サポートURL、問い合わせ先、販売者情報、特商法表記、プライバシー選択ページなど、必要な公開ページが用意されている

## 3. それ以外で人間がチェックする必要のあるもの

- [ ] App Storeのプライバシー詳細、プライバシーラベル、Google Playのデータセーフティ申告、プライバシーポリシー、実際のアプリ挙動・SDK挙動が一致している
- [ ] センシティブ権限や利用者情報の取得が、アプリ機能に対して必要最小限といえる
- [ ] IDFA、AAID、端末識別子、トラッキング、広告測定について、同意・オプトアウト・説明文言が法域ごとの要件を満たしている
- [ ] 子ども向け、または13歳未満の利用を実質的に想定するかどうかを、デザイン、文言、広告、カテゴリ、実ユーザー層から判断している
- [ ] 子ども向けまたは子ども利用の可能性がある場合、COPPA、年齢確認、保護者同意、子ども向けSDK制限、広告制限を満たしている
- [ ] 日本向けに提供する場合、個人情報保護法、消費者契約法、特定商取引法、資金決済法など、機能に応じた国内法の論点を確認している
- [ ] EU、英国、米国州法などの対象ユーザーがいる場合、GDPR、UK GDPR、CCPA/CPRA等の地域別プライバシー権利と同意要件を確認している
- [ ] 利用規約が、実際のサービス内容、課金、禁止事項、責任範囲、消費者保護規制と整合している
- [ ] ユーザー投稿、画像、文章、レビュー、プロフィール等を扱う場合、投稿コンテンツの権利帰属、利用許諾、削除、通報、ブロック、違法コンテンツ対応が妥当である
- [ ] アプリ名、アイコン、UI、文章、画像、音源、フォント、地図、データベース、APIレスポンスなどについて、自社権利または利用許諾が確認できている
- [ ] 他社アプリ、ブランド、OS、ストア、商標、著名キャラクターに誤認される表示やコピーキャット表現がない
- [ ] App Store / Google Playの課金ルールと、特商法などの販売表示義務が衝突しないように整理されている
- [ ] 広告、アフィリエイト、スポンサー表示、ネイティブ広告、リワード広告が、広告表示規制やプラットフォーム規約に照らして透明である
- [ ] ヘルスケア、金融、融資、投資、保険、ギャンブル、VPN、ニュース、医療、位置情報共有などの規制領域では、必要な免許、地域制限、追加開示、レビュー資料を確認している
- [ ] アプリ更新やSDK差し替えのたびに、プライバシーポリシー、ストア申告、プライバシーマニフェスト、データセーフティ、利用規約を再確認する運用が決まっている
- [ ] 高リスクな機能や国際展開前には、弁護士・プライバシー専門家による確認を受ける
